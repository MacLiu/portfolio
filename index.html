Face-Analysis (February 2014)

For CodeDay 2014, I developed a mobile app that rates users from a 1-10 scale based off of their facial features. This app uses a facial recognition API (Face++) that extracted landmarks and facial key-points from an image. Using these, I was able to develop my own algorithm to calculate the ratios and symmetry of the face and determined how far off that was from the Golden Ratio. The app won best mobile app at the event.

My Dining (Fall 2014)

For HackISU 2014, we reverse-engineered our school's flash-based online food ordering system ISU Webfood and built an iOS mobile application on top of it using Swift. We manages to have the whole application up and running by Sunday morning and even placed an order in their system. We took 3rd place of over 150 people!



Split Pay (Summer 2014)

For HackMidWest 2014, we created Simple Pay, an IOS+Android application that allows a user to create a bill for something, which his/her friends can submit payment for a percentage of the bill that they wish to pay. Once enough money is generated for the bill, the app gives all the money to the user who created this bill. Simple Pay is perfect for situations like throwing a party or wanting something for your birthday that is to expensive for one person to give. I was responsible for creating the IOS version of the application, while the others worked on the Android version. I created the entire front end expereince, integrated Parse API into my application for persistance to the cloud, and integrated Simplify API into the application for payment acceptance. My team was able to complete both the IOS and Android version of the application in under 24 hours. Our team ended up placing runner-up at the hackathon!



Who's Free (Summer 2014)

Have you ever tried making plans with a friends or family member just to find out that they already have plans? Who's Free is an IOS application that infroms you about who is free. To use Who's Free, a user simply sets their status as free or busy,and when there friends or family tries to message them or call the in the application, the application might restrict the user from taking this action based on the status. The pupose of this application is to allow the user to save time while trying to make plans or contact someone. It also infroms the user about which one of their friends are busy. I built the whole application from start to finish using Objective-C and Xcode for the development . I also used Parse's API for the application's persistence in the cloud.



PlacedIt (Summer 2014)

PlacedIt is a IOS application to help the user keep track of where he/she placed his/her items. To use PlacedIt, the user simply selects an item and mark down the location of where they placed their item. The user also has the option to take a picture of the item(at its placed location) and write down notes about it. I built PlacedIt entirely from scratch. For development, I used Objectiive-C, Xcode, MapKit Framework, and Core Data for persistence. For design, I used Adobe Photoshop CC.
